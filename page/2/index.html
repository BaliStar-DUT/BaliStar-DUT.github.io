MySQL基础（上）

学习Linux基础的笔记，视频教程来源慕课网：与MySQL的零距离接触

为第一到第五章部分

第一章 MySQL基本知识

1.启动/关闭MySQL

CMD中启动服务：
net start mysql
CMD中停止服务：
net stop mysql
所有Windows服务都可以通过上面命令启动与关闭

2.登录与退出

登录MySQL：
mysql [参数]

-D ,—database=name 打开指定数据库
-h 服务器名称
-p 密码

-P 端口号 MySQL默认端口号3306
-u 用户名
-V 输出版本信息并退出
—prompt=name 设置提示符

退出MySQL：
exit
quit
\q

3.修改MySQL提示符

MySQL默认提示符 mysql>
1.连接客户端时通过参数指定
shell>mysql -u root -p root —prompt 提示符
2.连接上客户端后，通过prompt命令修改
mysql>prompt 提示符

\D 完整的日期
\d 数据库名称
\h 主机名称
\u 当前用户名

4.MySQL常用命令

  1. 显示当前服务器版本
    SELECT VERSION();
  2. 显示当前日期时间
    SELECT NOW();
  3. 显示当前用户
    SELECT USER();
  4. 显示当前打开的数据库
    SELECT DATABASE();
  5. 显示错误信息
    SHOW WARNINGS;
  6. 显示数据库创建时的命令
    SHOW CREATE DATABASE db_name
    SHOW CREATE TABLE db_name;
  7. 以表格形式显示（直观）
    后面加”\G”;

  8. 5.操作数据库

  9. 创建数据库
    CREATE {DATABASE | SCHEMA} [IF NOT EXISTS] db_name [DEFAULT] CHARACTER SET [=] charset_name;
  10. 查看当前服务器中数据库列表
    SHOW {DATABASES | SCHEMAS} [LIKE ‘pattern’ | WHERE expr]
  11. 显示数据库创建时的命令
    SHOW CREATE DATABASE db_name
  12. 修改数据库
    ALTER {DATABASE | SCHEMA} [db_name] [DEFAULT] CHARACTER SET [=] charset_name;
  13. 删除数据库
    DROP {DATABASE | SCHEMA} [IF EXISTS] db_name;

    第二章 数据类型和操作数据表

    1.数据类型

  14. 整型 （有正负之分）

    TINYINT 1个字节
    有符号值：-128到127（-2^7到2^7-1）
    无符号值：0到255（0到2^8）
    SMALLINT 2个字节
    MEDIUMINT 3个字节
    INT 4个字节
    BIGINT 8个字节

  15. 浮点型

    FLOAT[(M,D)] M是数字总位数，D是小数点后面的位数。单精度浮点数精确到大约7位小数位
    DOUBLE[(M,D)]

  16. 日期时间型

    YEAR 1个字节
    TIME 3个字节
    DATE 3个字节
    DATETIME 8个字节
    TIMESTAMP 4个字节

  17. 字符型

    CHAR(M) M个字节，0<=M<=255
    VARCHAR(M) L+1个字节
    TINYTEXT L+1个字节
    TEXT L+2个字节
    MEDIUMTEXT L+3个字节
    LONGTEXT L+4个字节
    ENUM(‘value1’,’value2’,…)
    SET(‘value1’,’value2’,…) 元素做任意的排列组合

    2.数据表操作

  18. 创建数据表
    CREATE TABLE [IF NOT EXISTS] table_name(
     column_name data_type,
 .....
    )
    如果在CREATE语句后指定一个SELECT，MYsql将为在select中所有的单元创建新字段。
    可以用来复制表
    例如： create table weibotest select * from weibo where msg_ID=0
  19. 查看数据库中的数据表列表
    SHOW TABLES [FROM db_name] [LIKE ‘pattern’ | WHERE expr]
  20. 查看数据表结构
    SHOW COLUMNS FROM tb1_name
  21. 记录的插入与查找
    INSERT [INTO] tb1_name [(col_name,…)] VALUES (val,…)
    SELECT expr,… FROM tb1_name
  22. 空与非空
    NULL和NOT NULL
  23. 自动编号
    AUTO_INCREMENT
    • 自动编号，且必须与主键组合使用
    • 默认情况下，起始值为1，每次增量为1
  24. 主键约束
    PRIMARY KEY
    • 每张数据表只能存在一个主键
    • 主键保证记录的唯一性
    • 主键自动为NOT NULL
  25. 唯一约束
    UNIQUE KEY
    • 唯一约束可以保证记录的唯一性
    • 唯一约束的字段可以为空（NULL）
    • 每张数据表可以存在多个唯一约束
  26. 默认约束
    DEFAULT
    • 默认值
    • 当插入记录时，如果没有明确为字段赋值，则自动赋予默认值
  27. 删除数据记录
    DELETE FROM tb_name WHERE id = 3;

  28. 第三章 约束以及修改数据表

    1.外键约束的要求解析

    约束
  29. 约束保证数据的完整性和一致性
  30. 约束分为表级约束和列级约束

  31. 约束类型包括：

    • NOT NULL (非空约束)
    • PRIMARY KEY (主键约束)
    • UNIQUE KEY (唯一约束)
    • DEFAULT (默认约束)
    • FOREIGN KEY(外键约束)
      外键约束：实现一对一或一对多的关系

外键约束的要求：

  1. 父表和子表必须使用相同的存储引擎，而且禁止使用临时表；
  2. 数据表的存储引擎只能是InnoDB。
  3. 外键列和参照列必须具有相似的数据类型。其中数字的长度或是否有符号位必须相同；而字符的长度可以不同。
  4. 外键列和参照列必须创建索引。如果外键列不存在索引的话，MySQL将自动创建索引
    显示是否有创建索引
    SHOW INDEXES FROM tb_name\G;\G可以以图表的形式表现结果
    编辑数据表的默认存储引擎
    MySQL的配置文件：my.ini
    default-storage-engine=INNODB

    2.外键约束的参照操作

  5. CASECADE：从父表删除或更新且自动删除或更新子表中匹配的行
    FOREIGN KEY (pid) REFERENCES province(id) ON DELETE CASCADE;当从父表中删除记录的时候，子表中也会删除
  6. SET NULL：从父表删除或更新行，并设置字表中的外键列为NULL。如果使用该选项，必须保证字表列没有指定NOT NULL
  7. RESTRICT:拒绝对父表的删除或更新操作
  8. NO ACTION:标准SQL关键字，在MySQL中与RESTRICT相同

    3.表级约束和列级约束

    对一个数据列建立的约束，称为列级约束；
    对多个数据列建立的约束，称为表级约束；
    列级约束既可以在列定义时声明，也可以在列定以后声明；
    表级约束只能在列定义后声明。

    4.修改数据表-添加/删除列

  9. 添加单列
    ALTER TABLE tb_name ADD [COLUMN] col_name column_definition [FIRST | AFTER col_name]
  10. 添加多列
    ALTER TABLE tb_name ADD [COLUMN] (col_name column_definition,…)
  11. 删除列
    ALTER TABLE tb_name DROP col_name;

    5.添加约束

  12. 添加主键约束
    ALTER TABLE tb_name ADD [CONSTRAINT [symbol]] PRIMARY KEY index_type
  13. 添加唯一约束
    ALTER TABLE tb_name ADD [CONSTRAINT [symbol]] UNIQUE [INDES | KEY] [index_name] index_type
  14. 添加外键约束
    ALTER TABLE tb_name ADD [CONSTRAINT [symbol]] FOREIGN KEY [index_name] index_type reference_definition
  15. 添加/删除默认约束
    ALTER TABLE tb_name ALTER [COLUMN] col_name {SET DEFAULT literal | DROP DEFAULT};
  16. 删除主键约束
    ALTER TABLE tb_name DROP PRIMARY KEY;
  17. 删除唯一约束或索引
    ALTER TABLE tb_name DROP {INDEX | KEY} index_name;
    SHOW INDEXES FROM tb_name;//显示索引情况
  18. 删除外键约束
    ALTER TABLE tb_name DROP FOREIGN KEY fk_symbol;

    6.修改列定义和更名数据表

  19. 修改列定义
    ALTER TABLE tb_name MODIFY [COLUMN] col_name culumn_definition [FIRST | AFTER col_name];
  20. 修改列名称
    ALTER TABLE tb_name CHANGE [COLUMN] old_col_name new_col_name column_definition [FIRST | AFTER col_name];
  21. 数据表更名
    方法1：ALTER TABLE tb_name RENAME [TO | AS] new_tb_name;
    方法2：RENAME TABLE tb_name TO new_tb_name [,tb_name2 TO new_tb_name2]….

    第四章 操作数据表中的记录

    1.插入记录 INSERT

    INSERT [INTO] tb_name [(col_name,…)]{VALUES | VALUE}({expr | DEFAULT},…),(…),…
    INSERT users VALUES(NULL,’Tom’,’123’,25,1);
    INSERT users VALUES(DEFAULT,’Tom’,’123’,25,1);
    //默认自动编号的可以用NULL或DEFAULT来表示
    插入多个值：
    INSERT users VALUES(DEFAULT,’Tom’,’123’,25,1), (NULL,’Rose’,md5(‘123’),DEFAULT,0);
    md5()是mysql的一个计算哈希值的函数。

    2.插入记录 INSERT SET

    插入记录：INSERT [INTO] tb_name SET col_name={expr | DEFAULT},…
    说明：与第一种方式的区别在于，此方法可以使用子查询（SubQuery）,且一次只能插入一条
    INSERT users SET username=’Ben’,password=’4576 ‘;
    INSERT [INTO] tbname [(colname,…)] SELECT …
    说明：此方法可以将查询结果插入到制定数据表。
    INSERT test(usernaem) SELECT username FROM users WHERE age>=30;

    3.单表更新记录UPDATE

    单表更新：
    UPDATE [LOW_PRIORITY] [IGNORE] table_reference SET col_name1={expr1 | DEFAULT} [,col_name2={expr1 | DEFAULT} ]….[WHERE where_condition]
    UPDATE users age=age+5;//所有的age都增加5
    更新多列：
    UPDATE users SET age=age - id,sex=0;
    更新的时候加条件：
    UPDATE users SET age=age + 10 WHERE id%2 = 0;//对ID为偶数的记录更新age

    4. 单表删除记录DELETE

    DELETE FROM tb_name [WHERE where_condition];
    查询表达式解析：
    SELECT select_expr [,select_expr] [
    FROM table_reference
    [WHERE where_condition]
    [GROUP BY {col_name | positon} {ASC | DESC},….]
    [HVING where_conditon]
    [ORDER BY {col_name | expr | position} [ASC | DESC],…]
    [LIMIT {[offset,] row_count | row_count OFFSET offset}]
    ]

    5. 查询表达式解析

    查询表达式：
    每个表达式表示想要的一列，必须有至少一个。
    多个列之间用英文逗号分隔。
    星号()表示所有列。tb_name.可以 表示命名表的所有列。
    查询表达式可以使用[AS] alias_name 为其赋予别名。
    别名可用于GROUP BY.ORDER BY 或HAVING字句。

    6. WHERE语句进行条件查询

    WHERE语句条件查询：
    条件表达式：对记录进行过滤，在where语句中，可以使用mysql支持的函数或运算符。

    7. GROUP BY语句对查询结果分组

    GROUP BY语句对查询结果进行分组
    SELECT sex FROM users GROUP BY sex;

    8. HAVING语句设置分组条件

    [HVING where_]
    分组条件中的条件必须为聚合函数（max()，min()，avg()，count()，sum()）

    9. ORDER BY 语句对查询结果排序

    ORDER BY:对查询结果进行排序：
    [ORDER BY {col_name | expr | position} [ASC | DESC],…]
    SELECT * FROM users ORDER BY age , id DESC;//先按age升序排列，再按id降序排列

    10. LIMIT语句限制查询数量

    LIMIT语句限制查询数量：
    [LIMIT {[offset,] row_count | row_count OFFSET offset}];
    SELECT FROM users LIMIT 2;//返回前2条数据
    SELECT     FROM users LIMIT 3,2; //返回从下标3开始的2条数据；下标从0开始

    1.4 输出重定向

  3. 标准输入输出
    | 设备 | 设备文件名 | 文件描述符 | 类型 |
    | ——— | ——-: | :——: | :——- |
    | 键盘 | /dev/stdin | 0 | 标准输入 |
    | 显示器 | /dev/stdout| 1 | 标准输出 |
    | 显示器 | /dev/stderr| 2 | 标准错误输出 |

  4. 输出重定向

    > 标准输出重定向，单个大于号是覆盖
    >> 标准输出重定向，双大于号是追加
    2> 标准错误重定向，覆盖
    2>> 标准错误重定向，追加
    命令 > 文件 2>&1 已覆盖方式，标准错误和标准输出重定向到文件中
    命令 >> 文件 2>&1 以追加方式，
    命令 &> 文件 以覆盖方式，标准错误和标准输出重定向到文件中
    命令 &>> 文件 以追加方式，
    命令 >> 文件1 2>>文件2 把标准输出追加到文件1中，把标准错误追加到文件2中

  5. 输入重定向 < 小于号
    wc 统计行数，单词数，字符数
    -c 统计字节数
    -w 统计单词数
    -l 统计行数
    命令 < 文件 把文件作为命令的输入
    命令 << 标识符 双小于号 极少用

    1.5 管道符

    多命令顺序执行

  6. 命令1; 命令2 命令1成功与否，命令2顺序执行
  7. 命令1&&命令2 逻辑与，命令1成功，命令2才执行，否则不执行
  8. 命令1||命令2 逻辑或，命令1成功，命令2不执行，否则执行

    管道符

    命令1 | 命令2 命令1的正确输出作为命令2的操作对象
    netstat -an | grep “ESTABLISHED” 查看连接到服务器的情况

    通配符

  • ? 匹配一个任意字符
    • 匹配任意个字符
  • [] 匹配括号中任意一个字符
  • [-] 匹配括号中任意一个字符，-代表范围
  • [^] 逻辑非，表示匹配不是中括号内的一个字符
