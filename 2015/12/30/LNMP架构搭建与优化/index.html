LNMP架构搭建与优化

Dec 30, 2015 in linux

1.0 安装Mysql

  1. 安装Mysql二进制免编译包
  2. 下载：mysql到/usr/local/src/
    cd /usr/local/src/
    wget http://syslab.comsenz.com/downloads/linux/mysql-5.1.40-linux-i686-icc-glibc23.tar.gz
  3. 解压：
    tar zxvf /usr/local/src/mysql-5.1.40-linux-i686-icc-glibc23.tar.gz
    mv mysql-5.1.40-linux-i686-icc-glibc23 /usr/local/mysql
  4. 建立mysql用户
    useradd -s /sbin/nologin mysql
  5. 初始化数据库
    cd /usr/local/mysql
    mkdir -p /data/mysql ;chown -R mysql:mysql /data/mysql
    ./scripts/mysql_install_db –user=mysql –datadir=/data/mysql
  6. 拷贝配置文件及启动脚本
    cp support-files/my-large.cnf /etc/my.cnf
    cp support-files/mysql.server /etc/init.d/mysqld
    chmod 755 /etc/init.d/mysqld
  7. 修改启动脚本
    vim /etc/init.d/mysqld //修改“datadir=/data/mysql”
  8. 启动脚本加入系统服务项，设定开机启动mysql
    chkconfig –add mysqld
    chkconfig mysqld on
    service mysqld start

    1.1 php编译安装

  9. 下载php
    cd /usr/local/src
    wget http://am1.php.net/distributions/php-5.3.27.tar.gz
    tar zxvff php-5.3.27.tar.gz
  10. 创建相关用户
    useradd -s /sbin/nologin php-fpm
  11. 配置编译参数：cd php-5.3.27; ./configure
  12. 编译：make
  13. 安装 make install
  14. ps aux | grep php-fpm
  15. 修改配置文件
    cp php.ini-production /usr/local/php/etc/php.ini
    vim /usr/local/php/etc/php-fpm.conf
    写入如下内容：
    [global]
    pid = /usr/local/php/var/run/php-fpm.pid
    error_log = /usr/local/php/var/log/php-fpm.log
    [www]
    listen = /tmp/php-fcgi.sock
    user = php-fpm
    group = php-fpm
    pm = dynamic
    pm.max_children = 50
    pm.start_servers = 20
    pm.min_spare_servers = 5
    pm.max_spare_servers = 35
    pm.max_requests = 500
    rlimit_files = 1024
  16. 保存配置文件后，检查配置是否正确：
    /usr/local/php/sbin/php-fpm -t
  17. 启动php-fpm
    cp /usr/local/src/php-5.3.27/sapi/fpm/init.d.php-fpm /etc/init.d/php-fpm
    chmod 755 /etc/init.d/php-fpm
    service php-fpm start
  18. 开机启动
    chkconfig php-fpm on
  19. 查看进程：ps aux | grep php-fpm

    1.2 安装Nginx

    通过yum在线安装可以参考官方安装文档
  20. 下载nginx
    cd /usr/local/src/

  21. 解压:tar zxvf nginx-1.8.0.tar.gz

    ./configure –prefix=/usr/local/nginx –with=pcre
  23. 编译：make
  24. 安装:make install
  25. 编写nginx启动脚本，加入系统服务
    vim /etc/init.d/nginx
    脚本内容
    chmod 755 /etc/init.d/nginx
    chkconfg –add nginx
  26. 加入开机启动：chkconfig nginx on
  27. 更改nginx配置：vim /usr/local/nginx/conf/nginx.conf

  28. 检查配置文件是否有错：/usr/local/nginx/sbin/nginx -t
  29. 启动nginx ： service nginx start

    1.5 php-fpm配置文件

  42. php-fpm.conf 是php-fpm服务使用的配置文件；将原配置文件清空：> php-fpm.conf
  43. 复制php-fpm.cof的配置文件内容：
    php-fpm配置文件
  44. 多个pool可以通过不同的名字使用相同配置来实现
    启动多个pool可以使不同的域名使用不同的pool，通过在虚拟主机中设置fastcgi_pass 的路径来使用不同的pool socket。多个pool也可以设置不同的权限，通过不同的用户来控制不同的权限。
  45. 在pool中添加slowlog可以调试网站，例如记录执行超过2秒的脚本，这样就能知道网站访问速度瓶颈了。request_slowlog_timeout=1 记录超过1秒的脚本
  46. nginx中不同的pool也可以设置不同的open_basedir

    1.6 常见的502错误

  47. 首先查看nginx的错误日志
    发现sock无权限读取，发现socket的其他用户没有读写权限，不能使用php-fpm的sock

  48. 1.7 用户认证

  49. 用户认证用于给一些目录或文件加上访问权限，使拥有用户名和密码的才能访问
  50. 在nginx的配置文件中加入如下配置：
    `location ~ .*admin.php$ { 
    auth_basic "yangzhen auth";
auth_basic_user_file /usr/local/src/.htpasswd;//此为用户名与密码文件
    }`
  51. 生成用户名密码文件：利用apache的htpasswd命令来生成该文件：
    htpasswd -c /usr/local/src/.htpasswd yangzhen 然后输入密码即可生成加密的密码文件。
  52. 检查配置是否正确后重启：`nginx -t; service nginx reload
  53. 如果发现php文件不能被解析，则应将php文件中的相关配置复制进上面的配置中，以使php能被解析。`
  54. 可以用curl来验证：指定用户名密码的方法如下： curl -x localhost:8000 localhost:8000/index.html -u yang:yang
  55. 如果要给目录加限制，则改为：location /test/ 但是给单个文件加限制更为常见

    1.8 nginx域名重定向

  56. 将不同的域名导向同一个主机的作用。可以在配置文件中加入如下：
    if ( $host != 'www.test.com'){ rewrite ^/(.*)$ http://www.test.com/$1 permanent; }
  57. 后面的permanent表示永久重定向，可以参考论坛文档来详细了解
  58. 检查并重新加载配置文件：nginx -t; service nginx reload
  59. 域名重定向有助于搜索引擎找到真实对应的站点，使过时的域名都导入到新的域名中。

    1.9 nginx不记录指定文件类型日志

  60. 日志记录格式和位置都在配置文件中规定；access日志记录的访问资源的情况，像图片，css文件等是不需要记录的，可以配置实现。
  61. 在配置文件中添加：
    location ~ .*\.(gif|png|jpeg|swf|ico|css)$ { access_log off; }

    2.0 nginx日志切割

  62. 不像Apache可以用工具实现，需要我们自己写脚本实现
  63. 脚本内容如下：
    #! /bin/bash d =date -d “-1 day” +%F[ -d /tmp/nginx_log ] || mkdir /tmp/nginx_log mv /tmp/access.log /tmp/nginx_log/\$d.log /etc/init.d/nginx reload 2> /dev/null cd /tmp/nginx_log/ gzip -f $d.log
  64. 然后将这个脚本的执行加入到计划任务中，指定时间执行。

    2.1 nginx配置静态文件过期时间

  65. 配置静态文件缓存，如图片，css，js文件的过期时间配置
  66. 修改配置文件，在location中加入如下配置：
    `location ~ .*.(js|css){
    access_log off;
expires 2h;
    }`
  67. 在浏览器端，将看到：cache-control：max-age=毫秒数，添加过期时间将提高访问速度，节省带宽，不用每次请求都返回数据。

    2.2 nginx配置防盗链

  68. 依据reffer的来源来判断来源是否为本网站
  69. 在配置文件的location中加入如下配置：
    ` location ~ .*.(gif|jpeg|png|bmp|ico|swf|flv|rar|zip|gz|bz2|mp3|mp4)$ {
    access_log off;
expires 5h;
valid_referers none blocked  *.test.com;
if (\$invalid_referer){
    return 403;//此处也可将重定向到指定文件
}
    }`
  70. 检查配置文件后，重新加载配置文件。

    2.3 nginx访问控制

  71. nginx限制访问的方法，例如禁止某ip的访问，开放某些ip的访问。
  72. 在配置文件中加入如下配置：
    `location ~ .*admin.php$ { 
    allow 127.0.0.1;
deny all;
auth_basic "yangzhen auth";
auth_basic_user_file /usr/local/src/.htpasswd;//此为用户名与密码文件
    }`
  73. 检查配置文件后重新加载。
  74. 全局访问控制，在server字段下增加allow和deny；

    2.4 nginx禁止指定useragent

  75. 即限制指定客户端的访问
  76. 在配置文件中加入如下：
    ` if ($http_user_agent ~ ‘Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 6.1; WOW64; Trident/4.0; SLCC2; .NET CLR 2.0.50727; .NET CLR 3.5.30729; .NET CLR 3.0.30729; Media Center PC 6.0; InfoPath.3; .NET4.0C; .NET4.0E)’){
    return 403;
    }`
  77. 用curl 模拟指定useragent：
    curl -A "abcd" -xlocalhost:80 localhost/test/index.php -I //模拟useragent为abcd
  78. 不区分大小写匹配：~* 即表示不区分大小写匹配

    2.5 nginx代理详解

  79. 如果想访问google，可以通过代理来访问不能访问的网站。

  80. 在配置文件中加入：
    `server {
    listen 81;
    server_name www.facebook.com;

    location / {

    proxy_pass http://31.13.77.36/;
http://23.235.47.81/
proxy_set_header Host $host;

    }
    }` //表示将本机的访问导入到facebook的主页

  81. 如果想要代理多个网站，则可以改成：
    `upstream yang{
    server 23.235.47.81:80;
    server 23.235.47.81:80;
    server {
    listen 81;
    server_name www.facebook.com;

    location / {

    proxy_pass http://yang/;
proxy_set_header Host $host;//加上此行，否则可能502
#proxy_set_header X-Real_IP $remote_addr;//设定远程真实IP

    }
    }
    }`
